# User Personas - 테이블오더 서비스

## Overview
테이블오더 서비스의 주요 사용자 유형을 정의한 페르소나입니다. 각 페르소나는 시스템과 상호작용하는 방식, 목표, 동기, 불만사항을 포함합니다.

---

## Persona 1: 고객 (Customer)

### 기본 정보
- **이름**: 김민수
- **나이**: 28세
- **직업**: 회사원
- **역할**: 레스토랑 테이블에서 주문하는 고객

### 목표
- 빠르고 편리하게 메뉴를 탐색하고 주문하고 싶다
- 주문 내역과 상태를 실시간으로 확인하고 싶다
- 복잡한 로그인 절차 없이 즉시 주문하고 싶다
- 주문 실수를 최소화하고 싶다 (장바구니에서 확인 후 주문)

### 동기
- **편의성**: 직원을 부르지 않고 자유롭게 주문하고 싶다
- **시간 절약**: 대기 시간 없이 즉시 주문하고 싶다
- **정확성**: 메뉴 정보를 명확히 확인하고 주문하고 싶다
- **투명성**: 주문 상태를 실시간으로 확인하고 싶다

### 불만사항 (Pain Points)
- 직원을 부르기 어렵거나 대기 시간이 길다
- 메뉴판이 불명확하거나 정보가 부족하다
- 주문이 제대로 전달되었는지 불안하다
- 주문 상태를 알 수 없어 답답하다
- 추가 주문 시 매번 직원을 불러야 한다

### 사용 시나리오
1. 테이블에 앉아 태블릿을 켠다 (자동 로그인)
2. 메뉴를 카테고리별로 탐색한다
3. 원하는 메뉴를 장바구니에 추가한다
4. 수량을 조절하고 총 금액을 확인한다
5. 주문을 확정하고 주문 번호를 받는다
6. 주문 내역 화면에서 주문 상태를 확인한다
7. 추가 주문이 필요하면 2-6을 반복한다

## Persona 2: 매장 관리자 (Store Admin)

### 기본 정보
- **이름**: 박지영
- **나이**: 35세
- **직업**: 레스토랑 매니저
- **역할**: 매장 주문을 실시간으로 모니터링하고 관리하는 관리자

### 목표
- 모든 테이블의 주문을 실시간으로 모니터링하고 싶다
- 주문 상태를 효율적으로 관리하고 싶다
- 테이블 세션을 관리하고 과거 주문 내역을 조회하고 싶다
- 메뉴를 쉽게 추가/수정/삭제하고 싶다
- 주문 오류를 빠르게 수정하고 싶다

### 동기
- **효율성**: 인건비를 절감하고 운영 효율을 높이고 싶다
- **정확성**: 주문 누락이나 오류를 최소화하고 싶다
- **가시성**: 모든 테이블의 주문 상태를 한눈에 파악하고 싶다
- **유연성**: 메뉴를 빠르게 변경하고 관리하고 싶다
- **고객 만족**: 빠른 서비스로 고객 만족도를 높이고 싶다

### 불만사항 (Pain Points)
- 주문이 누락되거나 잘못 전달된다
- 여러 테이블의 주문을 동시에 관리하기 어렵다
- 주문 상태를 수동으로 추적해야 한다
- 메뉴 변경 시 메뉴판을 일일이 수정해야 한다
- 테이블 회전율을 파악하기 어렵다
- 과거 주문 내역을 찾기 어렵다

### 사용 시나리오
1. 관리자 시스템에 로그인한다
2. 실시간 주문 모니터링 대시보드를 확인한다
3. 신규 주문이 들어오면 알림을 받는다
4. 테이블별 주문 카드를 클릭하여 상세 내역을 확인한다
5. 주문 상태를 변경한다 (대기중 → 준비중 → 요리완료 → 전달완료)
6. 고객이 식사를 마치면 "이용 완료" 버튼을 클릭한다
7. 필요시 과거 주문 내역을 조회한다
8. 메뉴 관리 화면에서 메뉴를 추가/수정/삭제한다

## Persona 3: 슈퍼 관리자 (Super Admin)

### 기본 정보
- **이름**: 이준호
- **나이**: 42세
- **직업**: 레스토랑 체인 운영자
- **역할**: 여러 매장을 관리하고 매장 관리자 계정을 생성/관리하는 최고 관리자

### 목표
- 새로운 매장 관리자 계정을 생성하고 싶다
- 매장 관리자 계정을 활성화/비활성화하고 싶다
- 매장별 관리자 권한을 관리하고 싶다
- 시스템 전체를 안정적으로 운영하고 싶다

### 동기
- **확장성**: 새로운 매장을 쉽게 추가하고 싶다
- **보안**: 관리자 계정을 안전하게 관리하고 싶다
- **통제**: 매장별 접근 권한을 제어하고 싶다
- **효율성**: 관리자 계정 관리를 자동화하고 싶다

### 불만사항 (Pain Points)
- 매장 관리자 계정을 수동으로 생성해야 한다
- 퇴사한 관리자의 계정을 즉시 비활성화하기 어렵다
- 매장별 권한 관리가 복잡하다
- 관리자 계정 현황을 파악하기 어렵다

### 사용 시나리오
1. 슈퍼 관리자 시스템에 로그인한다
2. 새로운 매장 관리자 계정을 생성한다
   - 매장 식별자 (UUID) 할당
   - 사용자명 및 초기 비밀번호 설정
   - 권한 설정
3. 기존 매장 관리자 계정을 조회한다
4. 필요시 매장 관리자 계정을 비활성화한다
5. 매장 관리자 계정 목록을 확인한다

## Persona Mapping to User Stories

### 고객 (Customer) 관련 스토리
- 테이블 태블릿 자동 로그인
- 메뉴 조회 및 탐색
- 장바구니 관리
- 주문 생성
- 주문 내역 조회

### 매장 관리자 (Store Admin) 관련 스토리
- 매장 인증
- 실시간 주문 모니터링
- 주문 상태 변경
- 테이블 관리 (초기 설정, 주문 삭제, 세션 종료, 과거 내역 조회)
- 메뉴 관리

### 슈퍼 관리자 (Super Admin) 관련 스토리
- 매장 관리자 계정 생성
- 매장 관리자 계정 관리

---

## Summary

테이블오더 서비스는 3가지 주요 페르소나를 지원합니다:
1. **고객**: 빠르고 편리한 주문 경험을 원하는 레스토랑 방문객
2. **매장 관리자**: 효율적인 주문 관리와 실시간 모니터링을 원하는 매장 운영자
3. **슈퍼 관리자**: 여러 매장과 관리자 계정을 관리하는 최고 관리자

각 페르소나는 명확한 목표와 동기를 가지고 있으며, 시스템은 이들의 불만사항을 해결하고 목표를 달성할 수 있도록 설계되었습니다.
