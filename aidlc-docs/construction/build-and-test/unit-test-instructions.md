# Unit Test Instructions - Unit 4: SuperAdmin Frontend

## 테스트 환경

현재 Unit 4는 Standard 방식으로 생성되어 테스트 코드가 포함되지 않았습니다.

## 수동 테스트 체크리스트

### US-SA01: 슈퍼 관리자 로그인

1. **로그인 페이지 접근**
   - [ ] `http://localhost:5173/login` 접속
   - [ ] 로그인 폼 표시 확인

2. **로그인 성공**
   - [ ] Username: `superadmin`, Password: `super123` 입력
   - [ ] 로그인 버튼 클릭
   - [ ] 관리자 목록 페이지로 리다이렉트 확인
   - [ ] 헤더에 사용자명 표시 확인

3. **로그인 실패**
   - [ ] 잘못된 비밀번호 입력
   - [ ] 에러 메시지 표시 확인

4. **로그아웃**
   - [ ] 로그아웃 버튼 클릭
   - [ ] 로그인 페이지로 리다이렉트 확인

### US-SA02: 관리자 계정 생성

1. **계정 생성 모달**
   - [ ] "+ 계정 생성" 버튼 클릭
   - [ ] 모달 표시 확인
   - [ ] 매장 ID 자동 생성 확인

2. **계정 생성 성공**
   - [ ] 사용자명, 비밀번호 입력
   - [ ] 생성 버튼 클릭
   - [ ] 목록에 새 계정 추가 확인
   - [ ] 모달 닫힘 확인

3. **취소**
   - [ ] 취소 버튼 클릭
   - [ ] 모달 닫힘 확인

### US-SA03: 관리자 계정 조회

1. **목록 표시**
   - [ ] 관리자 목록 테이블 표시 확인
   - [ ] 매장 ID, 사용자명, 상태, 생성일, 액션 컬럼 확인

2. **검색/필터**
   - [ ] 검색창에 텍스트 입력
   - [ ] 필터링된 결과 확인

### US-SA04: 관리자 계정 비활성화

1. **비활성화**
   - [ ] 활성 상태 계정의 "비활성화" 버튼 클릭
   - [ ] 확인 팝업 표시 확인
   - [ ] 확인 클릭 후 상태 변경 확인 (활성 → 비활성)

### US-SA05: 관리자 계정 활성화

1. **활성화**
   - [ ] 비활성 상태 계정의 "활성화" 버튼 클릭
   - [ ] 확인 팝업 표시 확인
   - [ ] 확인 클릭 후 상태 변경 확인 (비활성 → 활성)

## 테스트 실행 방법

```bash
# 개발 서버 실행
cd table-order-superadmin-ui
npm run dev

# 브라우저에서 http://localhost:5173 접속
# 위 체크리스트 수동 테스트 진행
```

## 향후 자동화 테스트 추가 시
